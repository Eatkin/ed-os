`/home/ed/.claude/plans/robust-shimmying-hippo.md`. Summary:

- **Backend** (`helpers.js` + `ApiService.js`): add `updateTrajectory`,
  `updateMilestone`, `updateNote`, `updateLog`, `updateLootItem`,
  `updateCommitment` — same helper-mutates/ApiService-wraps-and-persists
  convention as every existing mutator. Editable fields kept scoped to
  "correcting mistakes": Trajectory/LootItem get full structural edits;
  Milestone/Note/Log/Commitment scoped to their text field (`text`,
  `note`/`trajectoryId`, `note`, `notes`) since their other fields either

  `CommitmentItem.js`/`LogItem.js`; a small always-enabled "✏️" icon

## 2026-07-23: Sort control on Goals screen

**File:** `components/goalsScreen/GoalsScreen.js` only.

Currently `active` trajectories are sorted by a hardcoded momentum
comparator with no way to change it. Plan:

1. Reuse the existing `components/shared/form/SelectButtons.js` (already
   used standalone in form fields — it just needs `value`/`onChange`,
   no dependency on `FormModal`) as a small "SORT BY" control rendered
   right under the "// GOALS" title, above the `ScrollView` (so it stays
   visible while scrolling), with three options: `MOMENTUM` (default,
   preserves current behavior), `NAME`, `LAST LOGGED`.
2. Add `const [sortBy, setSortBy] = useState("momentum")`.
3. Replace the inline `.sort((a, b) => {...})` with a small
   `SORT_COMPARATORS` map keyed by `momentum`/`name`/`lastLogged`, and
   call `Object.values(trajectories).sort(SORT_COMPARATORS[sortBy])`.
   - `momentum`: exactly the existing comparator (unchanged behavior).
   - `name`: `a.name.localeCompare(b.name)`.
   - `lastLogged`: same never-logged-sinks-to-bottom handling as
     momentum, just orders by `lastLoggedAt` descending with no
     momentum tiebreak.

No changes to archived trajectories (still just filtered, unsorted, as
today), no new context/global state — `sortBy` is local UI state to
this screen only.
